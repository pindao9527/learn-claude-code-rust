# Day 09: Review & Key Concepts

## 1. 并发模型：Tokio 异步任务
在 Rust 版 Agent 团队中，我们抛弃了原版的系统线程，选择了 `tokio::spawn`。
- **优势**：极低的内存占用（绿色线程），极高的 I/O 吞吐量。
- **注意**：传给 `tokio::spawn` 的闭包必须满足 `'static` 生命周期，因此所有外部变量（Client, API Key, Bus 等）都必须通过 `.clone()` 转移所有权。

## 2. 线程安全：Arc<Mutex<T>>
多 Agent 同时运行意味着会有多个线程尝试读写同一个“花名册” (`TeamConfig`)。
- **Arc (Atomic Reference Count)**：允许多个线程同时持有同一个数据的所有权。
- **Mutex (Mutual Exclusion)**：确保同一时间只有一个 Agent 能够修改数据。
- **最佳实践**：在 Rust 中，先获取 `lock()`，然后尽量缩小锁的持有时间。

## 3. 跨平台文件锁：fs2
原版 Python 在高并发写入信箱时存在数据错乱的风险。我们在 Rust 中引入了 `fs2` 库：
- **`lock_exclusive()`**：在写入 JSONL 消息前加独占锁。
- **`set_len(0)`**：在读取并消费信箱时，通过将文件长度置零来高效清空。
- **意义**：这保证了即便有成百上千个 Agent，信箱数据也绝对不会产生数据竞争（Data Race）。

## 4. LLM 思考逻辑：嵌套循环 (Auto-loop)
解决“慢半拍”问题的关键。
- **外层循环**：等待人类输入。
- **内层循环**：Agent 的连续思考（CoT）。只要模型决定调用工具，就立刻执行并自动进行下一轮请求，直到任务完成。
- **安全保护**：一定要设置最大循环次数（如 10 次或 30 次）并检查 `tool_calls` 是否为空，防止无限循环烧掉 API 额度。
