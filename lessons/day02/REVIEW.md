# Day 02 复习卡：所有权与类型系统

## 1. 所有权 (Ownership) —— Rust 的基石

为了管理内存，Rust 既没有垃圾回收（GC），也不需要手动释放内存，而是靠这三条规则：
1.  **所有者**: 每一个值都有一个变量作为它的所有者。
2.  **唯一性**: 同一时间只能有一个所有者。
3.  **释放**: 所有者离开作用域，内存立即自动回收。

### 移动 (Move) vs 克隆 (Clone)

- **Move**: 当执行 `let s2 = s1;` 时，所有权从 `s1` 移动到 `s2`，`s1` 废弃。
- **Clone**: 显式调用 `.clone()`，在堆上复制一份完整数据，`s1` 和 `s2` 都有效。
- **Copy**: 对于简单的“栈上数据”（如 `i32`, `bool`），赋值会自动复制，不发生 Move。

## 2. 借用与引用 (Borrowing & References)

不想交出所有权，只想让别人“用一下”？那就用借用。
- **不可变引用 `&T`**: 只读，可以有无数个。
- **可变引用 `&mut T`**: 可写，**同一时间只能有一个**，且不能与任何不可变引用共存。
  - **目的**: 彻底消灭数据竞争（Data Race）。

## 3. 强类型模型：枚举 (Enum) 与 结构体 (Struct)

- **枚举 (Enum)**: 描述“它可能是 A，也可能是 B”。
  - 在 `s02` 中，`Message` 是一个枚举，因为它可能是用户发的，也可能是系统发的。
- **模式匹配 (Match)**:
  - 它是“穷尽性”的，如果漏掉一种可能性，编译器就会骂你。
  - 使用 `if let` 处理只关心一种情况的简写。

## 4. 字符串的两种面孔

- `String`: 拥有所有权的字符串，可以修改。
- `&str`: 借用的字符串（切片），只读。
  - **实战建议**: 函数参数尽量使用 `&str`，这样既能接 `String` 也能接字面量，更灵活。

## 5. 序列化 (Serde)

- `#[serde(tag = "role", rename_all = "lowercase")]`:
  - 这是“相邻标签”模式。它将枚举转换成 JSON 时，会自动加上一个 `"role": "xxx"` 的 key。这让我们能够完美兼容外部 API 协议。
